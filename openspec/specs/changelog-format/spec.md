## changelog-format

### Requirements

1. `CHANGELOG.md` MUST exist at the project root
2. The file MUST follow Keep a Changelog format (https://keepachangelog.com/)
3. The file MUST have a `## [Unreleased]` section at the top (below the title)
4. Release sections MUST use the format `## [X.Y.Z] - YYYY-MM-DD`
5. Change entries MUST be categorized under: Added, Changed, Deprecated, Removed, Fixed, Security
6. The initial CHANGELOG.md MUST include an `[Unreleased]` section and a `[0.1.0]` section documenting the initial functionality
7. `CLAUDE.md` MUST include an instruction to add changelog entries when archiving changes
